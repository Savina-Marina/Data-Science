# Python: Package Management & Virtual Environments

## О проекте

В рамках данного учебного проекта была изучена работа с управлением зависимостями Python и изоляцией окружений. Основной акцент сделан на практическое применение инструментов для установки библиотек, парсинга данных, профилирования и тестирования кода.

Проект демонстрирует навыки работы с окружением разработки, сторонними библиотеками и анализом производительности Python-приложений.

---

## Реализованные задания

### Exercise 00 — Virtual Environment

Создано и настроено виртуальное окружение для изоляции зависимостей проекта.  
Реализован скрипт `venv.py`, который:

- определяет текущее виртуальное окружение через переменные среды;
- выводит путь к активному окружению;
- корректно обрабатывает запуск вне окружения.

Использованные технологии:
- `os`
- работа с переменными окружения
- виртуальные окружения (venv)

### Exercise 01 — Installing a package

В виртуальное окружение установлен пакет `termgraph`.

Реализовано:

- написан shell-скрипт `pies_bars.sh` для автоматизации запуска визуализации;
- подготовлен собственный датасет для отображения.

Использованные технологии:
- `termgraph`
- bash scripting

### Exercise 02 — Installing many libraries

Настроена установка зависимостей через `requirements.txt`.

Реализован скрипт `librarian.py`, который:
- проверяет запуск внутри виртуального окружения;
- устанавливает зависимости из requirements;
- выводит список установленных пакетов;
- сохраняет их в `requirements.txt`.

Использованные технологии:
- `pip`
- `requirements.txt`
- автоматизация установки зависимостей

### Exercise 03 — Web parsing (BeautifulSoup)

Реализован скрипт `financial.py` для парсинга финансовых данных с Yahoo Finance.

Функциональность:
- принимает тикер компании и название финансового показателя;
- извлекает данные из HTML-страницы;
- возвращает результат в виде tuple;
- реализована обработка ошибок (некорректный тикер или поле);
- добавлена задержка выполнения (`sleep`).

Использованные технологии:
- `requests` / HTTP-клиент
- `BeautifulSoup`
- парсинг HTML
- обработка исключений

### Exercise 04 — Profiling

Проведен анализ производительности скрипта `financial.py`.

Реализовано:
- использование `cProfile` для анализа времени выполнения;
- сравнение работы с и без `sleep`;
- оптимизация HTTP-запросов (альтернативный клиент);
- анализ количества вызовов функций;
- работа с `pstats` для получения агрегированной статистики.

Результаты сохранены в:
- `profiling-sleep.txt`
- `profiling-tottime.txt`
- `profiling-http.txt`
- `profiling-ncalls.txt`
- `pstats-cumulative.txt`

### Exercise 05. PyTest

- Директория для сдачи: `ex05/`
- Файлы для сдачи: `financial_test.py`
- Разрешённые функции: без ограничений

Скорость работы скрипта — не единственное, о чём стоит беспокоиться. Он может изначально работать не так, как вы задумали. Чтобы убедиться в корректности работы, необходимо провести модульное тестирование. Например, подать различные входные данные и проверить, соответствует ли результат ожидаемому.

1. В задании ex03 вы использовали одну или несколько функций. Для каждой функции создайте как минимум три теста с использованием библиотеки PyTest. Проверьте следующее:
    - Если запросить Total Revenue, возвращает ли скрипт выручку для указанного тикера?
    - Является ли возвращаемое значение кортежем (tuple)?
    - Если ввести некорректный тикер, возникает ли исключение?

2. Добавьте тесты в код вашего скрипта `financial.py`. Сохраните файл в директории как `financial_test.py`. Запустите PyTest. Все тесты должны успешно пройти. Если нет — доработайте скрипт.
